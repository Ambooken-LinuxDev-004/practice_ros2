                                    ################# URDF #################


URDF - Unified Robotics Description Format(it is saved as .xml file). It describes visual, physics, and connections of a robot with joints and links.

**** command to install urdf package - sudo apt-get install ros-humble-urdf-tutorial**** 

Link - used to describe a segment of a robot(defines visual, collision and inertial properties). Geometry and Origin are used in Visual and Collision properties to describe the Shape and Location of link.

NOTE:- To get accurate models, obtain model from any 3D Modelling Software(e.g Solidworks) and export the .stl files for the geometry.

EACH LINK CAN ONLY BE CONNECTED TO ONE PARENT WHEREAS EACH PARENT CAN HAVE MULTIPLE LINKS.

Types of geometry - Box(<box size="x y z"/>), Sphere(<sphere radius="r"/>), Cylinder(<cylinder radius="r" length="l"/>), Mesh(<mesh filename="file_path" scale="x y z"/>)

NOTE:- For Meshes the file types can be .stl or .dae files

Origin - define rotations(using yaw(z)-pitch(y)-roll(x)) and translations(using x, y, z in meters), relative to link's coordinate frame. Order of rotation for rpy is ZYX(first X, then Y and then Z).

NOTE:- The Rotation operation is applied first and then translation.

For visual and collision, Origin describes how geometry is transformed relative to link's frame and for inertial, Origin describes the location of center of mass relative to link's frame.

<origin rpy="r p y" xyz="x y z"/>

Axis colors:- X(Red), Y(Green), Z(Blue).

Material - used to define colors and transparency of color(using rgba).

<material name="color_name">
<color rgba="r g b a"/>
</material>

e.g red-rgba(1001), green-rgba(0101), blue-rgba(0011) etc..

Collision - describe bounding shape around link for collision detection.(use simplified shape for faster calibration).

Inertial - used to describe inertia[kg-m^2] about link's center of mass and mass[kg] of link.

By convention, we call this package whatever the name of the robot like this: "robotname_description"

#!/bin/bash - Add this at top

cleanup(){
  echo "Restarting ROS 2 daemon to cleanup before shutting down all processes..."
  ros2 daemon stop
  sleep 1
  ros2 daemon start
  echo "Terminating all ROS-2 related processes.."
  kill 0
  exit
} - created a function called cleanup. It stops and restarts ros2 daemon(within 1 second), which is useful for gazebo, rviz(simulations) as these simulation softwares might not shutdown properly after use(thus, it might block communication between node when executing..)

kill 0 - kills all processes and then we exit

trap 'cleanup' SIGINT - represents Ctrl + C, (when pressing ctrl + c instaed of stopping the process, it first execute cleanup() then it stops)..

                                ################# Coordinate Frames & Right-hand rule of Robotics #################


X-axis(red) - index finger (points forward and is positive direction)
Y-axis(green) - middle finger (points left and is positive direction)
Z-axis(blue) - thumb (points up and is positive direction)

Orientation(types of rotation):

Roll(x-axis) - ear touch shoulder
Pitch(y-axis) - chin touches chest
Yaw(z-axis) - shaking head no

World coordinate frame/Map frame - serves as global reference point for all robots and objects in a given environment
This never moves or changes(provide stable point of reference)

Placed in convinient place - cneter of room, battery charging station etc...(all other coordinate frames reference back to world frame)

Base frame - frame that's attached to robot's based(in mobile robot, the base frame moves(relative to the world frame)... along with robot)

Sensor frames - eyes/ears of robot. 

Joint frames - helps how how each joint bent/twisted

Origin - point where xyz(all perpendicular) meets

End-effector frame - coordinate frame at the very end of robotic arm. 

Odom - odometery(frame which starts to move)

Base_footprint frame - frame directly under the center of base link
